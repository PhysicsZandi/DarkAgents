# BACKEND COMPATIBILITY 

This document describes which backend code is implemented for each branch. 

## FOPT-PTA Branch

- `backend/semianalytic_pipeline.py`: this works only for conformal/classically scale-invariant single-field models, where the tree-level potential contains only a quartic term and no mass term, and the Coleman-Weinberg relation can be used to eliminate the quartic coupling in favor of couplings and the vev. It exploits the high-temperature expansion of the finite-temperature contribution to write down the effective potential in a quartic polynomial form, which allows to use analytic expressions for the bounce action. This approximation is proved to be valid not only at high temperatures, but also in the intended regime of strong supercooling, since the potential barrier is generated by thermal corrections and therefore is at small field / T values, which is exactly the regime where the high-temperature expansion is valid. Do not confuse with the name high-temperature expansion, which is a misnomer: it is an expansion in small field / T values, not in T itself. Furthermore, it uses a Gaussian approximation for the false-vacuum fraction and the percolation condition, which allows to speed up the scan over the parameter space, with great accuracy. This approximation is valid in every regime, since it is derived from first principles. 
Paper reference for this pipeline: arXiv:2602.02829 (Pascoli, Rosauro-Alcaraz, Zandi).

Selection rule:
- Use `backend/semianalytic_pipeline.py` only for conformal/classically scale-invariant single-field models with masses proportional to one scalar vev and no explicit tree-level mass term.
